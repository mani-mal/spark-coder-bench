# Publishing Plan

Target outputs for the benchmark, in order. **The framing is a systems + methodology case study,
not a model-ranking paper** (see `docs/findings/2026-07-02-audit-verification-and-decision.md`).

## 1. GitHub

- Public repo with this measurement/reporting code, the frozen spec, raw results, summaries,
  charts, and the final report.
- Clear README (done), reproducible methodology (`docs/methodology.md`), a **LICENSE** (MIT, added)
  and **CITATION.cff** (added).
- Link to the companion app repo (`taskflow-local-app-benchmark`) and its per-run tags
  (`baseline-v1..v6`). Note the known baseline-v4/v6 lineage break behind finding C1.
- Tag a release that matches the report version.

## 2. Written report

- `reports/dgx-spark-coding-model-benchmark-report.md` — the canonical write-up (**drafted**;
  systems + methodology framing, every table mapped to a source CSV + regeneration command, with
  the §3 coupling table, the C1 L2 before/after ablation, and the C2 L3 paired-subset correction).
  `reports/screenshots/` is still empty (optional; the architecture diagram at
  `docs/architecture/benchmark-architecture.jpg` is the main figure).
- Draw language **only** from the evergreen docs (`README`, `docs/HELP.md`,
  `docs/BENCHMARK_OVERVIEW.md`, `docs/methodology.md`) — they now tell one reconciled story.
- Sections: motivation; setup (DGX Spark GB10 + vLLM/TensorRT-LLM + OpenCode); the
  **serving-feasibility matrix** (no model serves on both runtimes — the headline systems result);
  methodology; results (L1 SWE-bench, L2 acceptance-check fraction — now **contract-visible** after
  the C1 rerun, with the before/after ablation as a headline methodology result; L3 LiveCodeBench
  pass@1 with the paired truncation analysis); the failure
  taxonomy + model-valid rates; measurement pitfalls (unified-memory, prompt-token-dominated tok/J,
  restart-truncated energy windows); limitations; conclusions.
- Every table/figure maps to a source CSV under `results/summary/` or `reports/charts/` **plus its
  regeneration command** (`analysis/aggregate-runs.py`, `analysis/figures.py`,
  `analysis/robust-summary.py`, `analysis/l3-conditional.py`).

## 3. arXiv

- Convert the report to LaTeX. Category: **cs.SE**, cross-list **cs.PF / cs.LG**. Realistic
  strength: technical report / experience paper (workshop-paper strength), not a main-conference
  empirical ranking paper.
- Emphasize the transparency posture: dated findings, raw retention, deterministic rescoring
  (`aggregate-runs.py` and `robust-summary.py` regenerate their tables byte-identically).
- Be explicit about limitations: single DGX Spark; **three models** across **three configurations**
  with model identity fully coupled to serving runtime/precision; single run per L1/L3 cell and
  L2 N = 20/8 (node/python) vs 4/4 (nemotron); protocol changed mid-collection; generated code
  executed without OS-level isolation on this kernel (disclosed). Descriptive claims only.
